# ImageDedup — 图片查重工具 功能介绍

## 软件简介

ImageDedup 是一款桌面图片查重工具，支持从文件夹、压缩包、Excel、PDF 中查找重复图片，并提供取证分析功能（检测盗图/篡改）。Windows 用户下载 EXE 即可使用，无需安装 Python。

---

## 一、支持的文件格式

| 类型 | 格式 |
|------|------|
| 图片 | JPG、JPEG、PNG、GIF、BMP、WebP、TIFF |
| 文档 | Excel（.xlsx/.xls）、PDF — 自动提取文档内嵌图片 |
| 压缩包 | ZIP、RAR、7z、tar、gz、bz2、tgz — 支持嵌套压缩包递归解压 |

---

## 二、三种查重方式

### 1. 精准匹配（MD5）
- 通过文件 MD5 值判断是否完全一致
- 适用于：复制粘贴的完全相同文件

### 2. 感知哈希（pHash）
- 通过图片视觉内容计算哈希值，容忍轻微差异
- 可调节相似度阈值（默认 10，范围 0~30，越小越严格）
- 适用于：缩放、轻微裁剪、格式转换后的图片

### 3. 特征匹配（ORB）
- 通过 ORB 关键点检测图片局部特征
- 适用于：大幅裁剪、旋转、局部重叠的图片

> 三种方式可同时勾选，扫描结果会分组显示。

---

## 三、手机截图一机多号检测

针对同一手机多个账号的截图（主体内容相同，但顶部状态栏的时间、电量、信号不同），软件会：
- 单独计算顶部 8% 区域的哈希值
- 综合全图（80%权重）和顶部（20%权重）的差异计算相似度
- 自动标记 `[一机多号截图]`

---

## 四、取证分析

双击查重结果中的分组可打开取证对比窗口，包含 5 个分析维度：

| 分析项 | 说明 |
|--------|------|
| 元数据对比 | 对比两张图的 EXIF 信息（相机型号、拍摄时间、GPS 等） |
| 像素级差异 | 逐像素对比，生成差异热力图，显示平均/最大差异值 |
| ELA 错误级别分析 | 检测图片是否被编辑过（高亮区域 = 可能被篡改） |
| 噪声分析 | 提取图片噪声特征，判断是否来自同一设备 |
| 光照与边缘检测 | 对比光照直方图和边缘轮廓，检测透视/光照差异 |

双击单张图片可打开轻量查看器，支持单图的 ELA、噪声、光照分析。

## 五、压缩包解压

### 解压路径
```
~/.cache/ImageDedup/extracted/
```
- Windows 实际路径：`C:\Users\你的用户名\.cache\ImageDedup\extracted\`
- macOS 实际路径：`/Users/你的用户名/.cache/ImageDedup/extracted/`

### 解压行为
- 拖入或点击"添加压缩包"后自动解压到上述目录
- 支持嵌套压缩包递归解压（压缩包里的压缩包也会自动解压）
- 解压后的文件永久保留，不会自动清理
- 下次启动软件时自动加载已解压的目录
- 同名压缩包会自动追加 `_1`、`_2` 后缀，不会覆盖

> 如需释放磁盘空间，手动删除上述目录中不需要的文件夹即可。

## 六、界面操作

### 添加来源
- 点击「添加文件夹」「添加文件」「添加压缩包」按钮
- 或直接拖放文件/文件夹/压缩包到左侧来源列表
- 来源以树形结构展示，可勾选/取消勾选

### 扫描与结果
- 点击「开始扫描」，进度条实时显示扫描阶段
- 增量扫描：显示本次新增、修改、删除的文件数量
- 结果树按分组展示，显示检测方式、相似度、文件大小、图片尺寸

### 批量操作
- 「全选重复项」— 每组保留最优文件，自动勾选其余文件
- 「批量删除选中」— 一键删除所有勾选的重复文件
- 右键菜单：从结果中移除 / 删除文件（不可恢复）

### 筛选与排序
- 文件名搜索
- 按检测方式筛选（精准/感知/特征）
- 按相似度范围筛选（0~100%）
- 点击列头排序

### 报告导出
- 点击「生成 HTML 报告」导出自包含的 HTML 文件
- 报告含缩略图、分组信息、相似度，可在浏览器中查看

---

## 七、设置

点击「设置」按钮打开设置对话框，分三个页签：

| 页签 | 可调参数 |
|------|---------|
| 基础 | 并行线程数、感知哈希阈值、日志级别 |
| 高级 | ORB 特征数、匹配比率、最小匹配分数、顶部裁剪比例、混合权重 |
| 取证 | ELA 压缩质量、放大倍数、噪声 sigma、Canny 边缘阈值、最大分辨率 |

设置保存在 `~/.config/image_dedup/settings.json`，可随时重置为默认值。

---

## 八、缓存

- 缓存路径：`~/.cache/image_dedup/cache.db`（SQLite）
- 已计算的哈希值会缓存，相同文件不会重复计算
- 超过 30 天的扫描记录自动清理
- 文件修改后自动重新计算哈希
